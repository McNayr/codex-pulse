# Session Shutdown

Use this before ending a Codex session.

## Required Closeout

1. Update `CURRENT_STATE.md` if reality changed.
2. Update `SESSION_SAVE.md` or `RESUME.md` with the exact restart point.
3. Update `TODO.md` with the next concrete actions.
4. Append to `CHRONOLOGY.md`.
5. Capture reusable learning in a lessons, decisions, or recurring-problems
   file if the session got smarter.
6. Update the project brief only if the high-level summary changed.

## Minimum Save Payload

Every session should leave behind:

- what changed
- what was verified
- what is blocked
- the exact next step
- which files to open first next time

## Shutdown Rule

If the next session would need terminal history to reconstruct state, the
session was not closed correctly.
